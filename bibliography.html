Research and Technical References

Annotated Bibliography

These sources align with the technical themes documented in the Ozora portfolio. The bibliography focuses on references that were directly relevant to the repository's implementation choices, including SwiftUI app structure, WebSocket communication, Express middleware, JSON-backed persistence, system iconography, and the Mermaid diagram format used to document the capstone artifacts.

Architecture

These sources supported the architectural split between the SwiftUI client, the Express backend, and the real-time update layer. They were especially relevant for understanding how the iOS app could connect to local services during development and how the backend could expose lightweight middleware-driven routes.

  1. Apple Developer. “webSocketTask(with:protocols:).” Apple Developer Documentation, https://developer.apple.com/documentation/foundation/urlsession/websockettask%28with%3Aprotocols%3A%29.
  2. Express.js. “body-parser.” Express Resources, https://expressjs.com/en/resources/middleware/body-parser.
  3. Express.js. “Installing.” Express Getting Started Guide, https://expressjs.com/en/starter/installing.html.
  4. Socket.IO. “Socket.IO.” Socket.IO, https://socket.io/.

UI and Ordering Flow

These references shaped the mobile interface and interaction model. They were useful for organizing the app around navigation, tab-based structure, state-driven updates, and system-native visual language that matched the iOS environment.

  1. Apple Developer. “Human Interface Guidelines.” Apple Developer Documentation, https://developer.apple.com/design/human-interface-guidelines/.
  2. Apple Developer. “Managing User Interface State.” Apple Developer Documentation, https://developer.apple.com/documentation/swiftui/managing-user-interface-state/.
  3. Apple Developer. “NavigationView.” Apple Developer Documentation, https://developer.apple.com/documentation/swiftui/navigationview.
  4. Apple Developer. “SF Symbols.” Apple Developer, https://developer.apple.com/sf-symbols/.
  5. Apple Developer. “Tab.” Apple Developer Documentation, https://developer.apple.com/documentation/SwiftUI/Tab.

Data Model

These sources informed how Ozora represented data across the backend and the SwiftUI client. They were relevant to the use of JSON-backed records, stable identifiers, and state-driven model updates that could feed both live UI and stored history.

  1. Apple Developer. “Model Data.” Apple Developer Documentation, https://developer.apple.com/documentation/SwiftUI/Model-data.
  2. Node.js. “File System.” Node.js Documentation, https://nodejs.org/api/fs.html.
  3. npm Docs. “package.json.” npm Docs, https://docs.npmjs.com/cli/v9/configuring-npm/package-json.
  4. uuidjs. “uuid.” npm, https://www.npmjs.com/package/uuid.

Authentication and Access

These sources were relevant to Ozora's prototype access model, which uses a mock login flow, persisted local state, local development networking, and permissive CORS settings so the iOS app and admin dashboard can communicate with the backend.

  1. Apple Developer. “AppStorage.” Apple Developer Documentation, https://developer.apple.com/documentation/SwiftUI/AppStorage?changes=_2.
  2. Apple Developer. “Preventing Insecure Network Connections.” Apple Developer Documentation, https://developer.apple.com/documentation/Security/preventing-insecure-network-connections.
  3. expressjs. “cors: Node.js CORS Middleware.” GitHub, https://github.com/expressjs/cors.

Testing and Revision

These sources supported the way the final portfolio documents iteration. Mermaid made it practical to produce architecture and workflow artifacts from code-backed themes, while Node's HTTP documentation aligned with the simulator scripts that drive repeatable demo validation in the Ozora backend.

  1. Mermaid. “Flowcharts - Basic Syntax.” Mermaid, https://mermaid.js.org/syntax/flowchart.html.
  2. Mermaid. “Sequence Diagrams.” Mermaid, https://mermaid.js.org/syntax/sequenceDiagram.html.
  3. Node.js. “HTTP.” Node.js Documentation, https://nodejs.org/api/http.html.
